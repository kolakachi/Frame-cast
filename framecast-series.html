Series Bible
The persistent context the AI receives every time it generates an episode. The more specific you are, the more consistent every episode will be.
⚡
How the AI uses this
The bible is injected as a system prompt prefix before every episode generation. It combines with the rolling episode summary chain so the model always knows the show format, tone, audience, and recurring rules — without you repeating it per episode.

Episode memory & continuity
⟳
How episode memory works
After each episode is marked complete, Framecast auto-generates a ~150 word summary and stores it. When the next episode is created, the last N summaries are injected as context — so the AI knows what was covered, avoids repetition, and can reference previous episodes naturally.

Step 3 of 5
Characters
Define recurring personas that appear in your series. Character descriptions are injected into visual prompts to keep faces, style, and presence consistent across episodes.
☻
How character descriptions are used
Personality notes get injected into the script generation prompt — so the narrator's voice or a character's dialogue stays true to how you defined them. Visual descriptions are prepended to every AI image and scene matching prompt that involves this character.
