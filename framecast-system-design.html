Framecast — System Design & Cost Economics

AI Video Publishing Platform for Faceless Creators · Architecture, Infrastructure, and Unit Economics

1. High-Level Architecture

The system is designed as an async-first, queue-driven pipeline. Video generation is not real-time — the user submits a project, the system processes it through multiple AI stages, assembles the result, renders to video, and delivers. This lets us optimize for throughput and cost rather than latency.

7. Cost Optimization Strategies

Phase 1 — MVP Launch

  • Use GPT-4o-mini for all LLM tasks — cheapest, fast enough for script gen ($0.15/1M input tokens)
  • Pexels free tier for stock footage — no per-clip cost, decent quality
  • ElevenLabs Scale plan — volume pricing kicks in at $0.18/1K chars
  • Self-host Whisper on a single T4 GPU — $0.40/hr on RunPod, handles ~500 alignments/day
  • Hetzner dedicated servers for render — 80% cheaper than AWS EC2 for CPU-bound FFmpeg
  • Cloudflare R2 over AWS S3 — zero egress charges saves hundreds/mo at scale

Phase 2 — Scale (>10K renders/mo)

  • Introduce open-source TTS tier (Fish Speech / Coqui XTTS) for Creator plan — cuts TTS COGS by 80%
  • Cache reusable audio segments — CTAs, intros/outros don't need re-generation
  • Smart stock footage caching — hash scene descriptions, reuse matched clips across projects
  • Spot instances for render workers — non-urgent batch renders on spot/preemptible, 60-70% savings
  • Negotiated ElevenLabs enterprise pricing at $0.10-0.12/1K chars

Phase 3 — At Scale (>100K renders/mo)

  • Self-hosted TTS models on dedicated GPU fleet — drops TTS cost to ~$0.01-0.02/video
  • Pre-rendered template components — render common intro/outro/CTA blocks once, composite at assembly
  • Regional render farms — co-locate render workers near heavy-use geographies
  • Custom fine-tuned small LLM for script gen — lower latency, lower cost than API calls

Stock footage API limits / quality Medium Multi-provider fallback (Pexels → Pixabay → Storyblocks); allow user uploads; AI image generation as alternative
Render compute spikes (batch exports) Medium Queue priority by plan tier; auto-scale render workers; rate limit free tier; pre-warming for predictable loads
Copyright claims on stock content Medium Only use CC0/royalty-free sources; display license info; user accepts terms per export

10. Summary

Framecast is architecturally a queue-driven AI pipeline with a thin editor UI. The core business challenge is managing per-video COGS (dominated by TTS at ~$0.11/video) while charging enough per render to maintain 70%+ gross margins. The technical moat comes from template systems, caption quality, and variant generation speed — not from any single AI capability.

Bottom line: At $0.15 COGS and $0.50 effective revenue per video, with $400/mo infrastructure, you break even at ~1,200 renders/month (≈30 Studio-plan customers). The economics improve significantly with scale as infrastructure costs are largely fixed and TTS volume pricing kicks in.
